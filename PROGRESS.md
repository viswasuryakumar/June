# PROGRESS.md

This file is the append-only progress log for the repository.

## Format

Each new entry is a level-3 heading, then two short blocks with a blank line
between them and around the heading. Keep both blocks concise: long enough
that someone asking "how did you implement this?" can answer from the
technical block alone, short enough to skim. Skip padding and
self-justification.

    ### YYYY-MM-DD — Title

    **What happened:** plain-language summary — what changed and why,
    understandable without reading any code.

    **How it was implemented:** the technical detail — files/functions
    touched, the actual mechanism, key values — followed by a short code
    snippet showing the core part (not the whole file/diff):

    ```language
    the few lines that actually matter
    ```

    **Note / Open items:** anything else worth flagging (omit if none).

Entries before 2026-07-13 predate this format and are left as-is — this file
is append-only, so historical entries are never rewritten, only superseded by
new ones. New entries always go at the true end of the log, in the order
they happened. (Proposed as a WORKFLOW.md §5 format update for Codex to
apply.)
